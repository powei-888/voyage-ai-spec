# 07. Roadmap

## v0.1 - Foundation MVP

Goal: prove the core workspace model.

Scope:

- Trip CRUD
- Member management
- Day-by-day timeline
- Event CRUD
- Manual expense creation
- Equal split calculation
- Receipt upload with mock OCR
- Receipt confirmation flow
- Booking CRUD
- AI proposal entity and basic proposal lifecycle

Exit criteria:

- A user can create a trip and invite members.
- A user can create daily itinerary events.
- A user can record shared expenses.
- A user can upload a receipt and confirm it into an expense.
- A user can view member balances.
- AI proposal data model exists.

## v0.2 - Practical Travel Workspace

Goal: make the product useful for a real trip.

Scope:

- Better dashboard
- Receipt list and pending review queue
- Booking hub improvements
- Link booking to event
- Link expense to event
- Basic comments or notes
- Better mobile UI
- Basic itinerary conflict check

## v0.3 - AI Assistance

Goal: make AI useful beyond chat.

Scope:

- AI itinerary check
- AI itinerary relaxation proposal
- AI expense category suggestion
- Budget summary
- Booking parser draft flow
- Structured proposal preview UI

## v0.5 - Travel Memory

Goal: make the product valuable after the trip ends.

Scope:

- Photo upload
- Photo to event linking
- Daily memory draft
- Trip replay page
- Shareable private memory page

## v1.0 - Public Product Candidate

Goal: prepare for real users.

Scope:

- Invite links
- Better permission system
- Production OCR provider
- Production AI provider abstraction
- Queue worker deployment
- Object storage integration
- Activity logs
- Docker deployment
- Basic observability

## Future Ideas

- Travel Inbox for PDFs, emails, screenshots, and booking documents
- Offline mode for timeline and bookings
- Live location sharing
- Calendar integration
- Advanced split methods
- Multi-currency support
- AI travel replay video
- One-year time capsule reminder
- Public itinerary templates
- Plugin system
